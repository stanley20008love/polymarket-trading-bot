# Polymarket 量化交易系统

100U起步的Polymarket自动交易机器人，Python全栈，支持模拟/实盘切换。

## ⚡ 快速开始

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置钱包
cp .env.example .env
# 编辑 .env 填入你的钱包信息

# 3. 先跑测试 (不需要钱包)
python3 test_system.py

# 4. 模拟模式运行 (DRY_RUN=true, 不花真钱)
python3 bot.py

# 5. 确认一切正常后切实盘
# 编辑 .env: DRY_RUN=false
python3 bot.py
```

## 🎯 三大策略

### 1. YES+NO 套利
当 YES价格 + NO价格 < 1 - 手续费 时，同时买入双方锁定差价利润。
- 实际上这种机会非常稀少（市场效率高）
- 即使出现，空间通常只有1-2%
- 需要同时成交双方，存在部分成交风险

### 2. 均值回归
当市场YES价格 < 10% 或 > 90% 时，买入便宜的一方。
- 逻辑：极端定价可能存在错误
- 风险：极端价格可能就是正确的（市场比你知道的多）
- 100U资金下每次只投10-20U

### 3. 事件驱动
高成交量+价格异动时，反向操作赌过度反应。
- 最低风险策略，只做价格极端的市场
- 100U下此策略仓位减半
- 置信度最低

## 🛡️ 风控硬规则

| 规则 | 参数 | 说明 |
|------|------|------|
| 止损 | 5% | 单笔亏损达5%自动平仓 |
| 止盈 | 15% | 单笔盈利达15%自动平仓 |
| 日亏损限制 | 3% | 日亏损达3%触发熔断 |
| 周亏损限制 | 8% | 周亏损达8%触发熔断 |
| 最大持仓 | 3个 | 满了不开新仓 |
| 单笔上限 | 20%资金 | 防止重仓 |
| 最小下单 | 5U | Polymarket最小下单约5U |

## 💰 100U本金的现实

坦白讲，100U在Polymarket做量化非常紧张：

| 项目 | 实际值 |
|------|--------|
| Polymarket最小下单 | 约5U |
| Polygon Gas费 | $0.01-0.05/笔 |
| 手续费 | Taker 1% (有流动性时Maker 0%) |
| 套利空间 | 通常0-2%，100U单笔赚0-2U |
| 均值回归 | 潜在回报高但风险大 |

**建议操作流程**：
1. 先模拟跑48小时，观察策略信号质量
2. 用10U实盘测试1周
3. 确认策略有效后逐步加到100U
4. 盈利稳定后考虑追加到1000U

## 🔐 钱包配置指南

### 获取私钥和Funder地址

1. 登录 [Polymarket](https://polymarket.com)
2. 进入 Settings → Security → Export Wallet
3. 复制 Private Key (0x开头的64位)
4. 复制你的 Wallet Address
5. 如果你用Email/Magic登录：
   - Funder Address = Polymarket分配的Proxy地址（在账户页面查看）
   - SIGNATURE_TYPE=1
6. 如果你用MetaMask等浏览器钱包：
   - Funder Address = 你的钱包地址
   - SIGNATURE_TYPE=0

### 安全提醒

- ⚠️ **永远不要把私钥提交到GitHub**
- ⚠️ **先跑DRY_RUN=true确认无误**
- ⚠️ **小额试水，不要一上来就全仓**
- ⚠️ **确保.env在.gitignore中**

## ❓ 常见问题

**Q: 为什么没有套利机会？**
A: Polymarket是连续竞价(CLOB)，市场效率很高。YES+NO价格通常接近1.0。套利机会偶尔出现但转瞬即逝。

**Q: 均值回归策略安全吗？**
A: 不安全。极端价格可能就是正确的。比如"2025年BTC到50万"定价5%，很可能真的不会发生。需要你对市场有独立判断。

**Q: 100U能赚钱吗？**
A: 很难。扣除Gas和手续费后，利润空间很小。建议把系统当作学习工具，用模拟模式积累经验，等策略验证后再追加资金。

**Q: 和V26仪表盘有什么区别？**
A: V26是诊断仪表盘（前端展示），这个是实际交易程序（后端执行）。两者独立运行。
